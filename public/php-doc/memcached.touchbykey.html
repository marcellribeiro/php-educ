Memcached::touchByKey

(PECL memcached >= 2.0.0)

Memcached::touchByKey — Define uma nova expiração para um item em um servidor específico

Descrição

public Memcached::touchByKey(string $server_key, string $key, int $expiration): bool

Memcached::touchByKey() é funcionalmente equivalente a Memcached::touch(), exceto que a forma livre server_key pode ser usada para mapear a key para um servidor específico.

Parâmetros

server_key

A chave que identifica o servidor para armazenar o valor ou de onde recuperá-lo. Em vez de fazer o hash na chave real do item, o hash é feito na chave do servidor ao decidir com qual servidor memcached se comunicar. Isso permite que itens relacionados sejam agrupados em um único servidor para eficiência com múltiplas operações.

key

A chave sob a qual armazenar o valor.

expiration

O tempo de expiração padrão é 0. Veja Tempos de Expiração para mais informações.

Valor Retornado

Retorna true em caso de sucesso ou false em caso de falha. Use Memcached::getResultCode() se necessário.
