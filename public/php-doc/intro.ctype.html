Introdução

As funções providas por esta extensão verificam se um caractere ou uma string está dentro de uma certa classe de caracteres de acordo com o atual locate (veja também setlocale()).

Quando chamada com um argumento inteiro, estas funções comportam-se exatamente como ocorro na linguagem C usando ctype.h. Significa que se você passar um inteiro menor que 256, será usado o valor ASCII dele para ver se está dentro do intervalo especificado (dígitos estão entre 0x30-0x39). Se o numero está entre -128 e -1 (incluindo) então 256 será adicionado e a verificação está concluida.

Aviso

A partir do PHP 8.1.0, passar um argumento que não seja string tornou-se obsoleto. No futuro, o argumento será interpretado como uma string em vez de um ponto de código ASCII. Dependendo do comportamento pretendido, o argumento deve ser convertido para string ou uma chamada explícita para chr() deve ser feita.

Quando chamadada com uma string como argumento elas irão verificar todos caracteres na string e irá somente retornar true se todos caracteres da string combinam com o requisitado critério. Quando chamada com uma string vazia, o resultado sempre será false.

Passando qualquer tipo além de string ou inteiro será retornado imediatamente false.

Deve ser notado que funções ctype são sempre preferência sobre expressões regulares, e as equivalentes funções str_* e is_*. Isto pelo fato de que ctype usa a biblioteca nativa C e assim processar significantemente mais rápido.

Nota:

Essas funções não são relacionadas as da biblioteca "ctypes" do Python. O nome da extensão vem do arquivo de cabeçalho ctype.h C que seus equivalentes C estão definidos.

Esta extensão também é anterior a biblioteca "ctypes" do Python, então qualquer confusão causada por conta da nomenclatura dificilmente é por conta do PHP.
