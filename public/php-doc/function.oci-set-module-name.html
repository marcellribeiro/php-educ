oci_set_module_name

(PHP 5 >= 5.3.2, PHP 7, PHP 8, PECL OCI8 >= 1.4.0)

oci_set_module_name — Sets the module name

Descrição

oci_set_module_name(resource $connection, string $name): bool

Sets the module name for Oracle tracing.

The module name is registered with the database when the next 'round-trip' from PHP to the database occurs, typically when an SQL statement is executed.

The name can subsequently be queried from database administration views such as V$SESSION. It can be used for tracing and monitoring such as with V$SQLAREA and DBMS_MONITOR.SERV_MOD_ACT_STAT_ENABLE.

The value may be retained across persistent connections.

Parâmetros

connection

Um identificador de conexão Oracle, retornado por oci_connect(), oci_pconnect() ou oci_new_connect().

name

User chosen string up to 48 bytes long.

Valor Retornado

Retorna true em caso de sucesso ou false em caso de falha.

Exemplos

Exemplo #1 Setting the module name

<?php

$c = oci_connect('hr', 'welcome', 'localhost/XE');

// Record the module
oci_set_module_name($c, 'Home Page');

// Code that causes a round-trip, for example a query:
$s = oci_parse($c, 'select * from dual');
oci_execute($s);
oci_fetch_all($s, $res);

sleep(30);
?> 
// While the script is running, the administrator can see the
// modules in use:

sqlplus system/welcome
SQL> select module from v$session;

Notas

Nota: Requisitos de versão do Oracle

Esta função estará disponível quando o PHP for compilado com as bibliotecas do Oracle Database a partir da versão 10g em diante.

Dica

Desempenho

Com versões anteriores do OCI8 ou do Oracle Database, as informações do cliente podem ser definidas usando o pacote Oracle DBMS_APPLICATION_INFO. Isso é menos eficiente do que usar oci_set_client_info().

Cuidado

Pegadinha da Viagem de Ida e Volta

Algumas mas não todas as funções do OCI8 causam viagens de ida e volta. Viagens de ida e volta ao banco de dados podem não ocorrer com consultas quando o cache de resultados está habilitado.
