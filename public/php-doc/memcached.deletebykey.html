Memcached::deleteByKey

(PECL memcached >= 0.1.0)

Memcached::deleteByKey — Remover um item de um servidor específico

Descrição

public Memcached::deleteByKey(string $server_key, string $key, int $time = 0): bool

Memcached::deleteByKey() é funcionalmente equivalente a Memcached::delete(), exceto que a forma livre server_key pode ser usada para mapear a key para um servidor específico.

Parâmetros

server_key

A chave que identifica o servidor para armazenar o valor ou de onde recuperá-lo. Em vez de fazer o hash na chave real do item, o hash é feito na chave do servidor ao decidir com qual servidor memcached se comunicar. Isso permite que itens relacionados sejam agrupados em um único servidor para eficiência com múltiplas operações.

key

A key a ser excluída.

time

A quantidade de tempo que o servidor aguardará para excluir o item.

Nota: A partir da memcached 1.3.0 (lançada em 2009), este recurso não é mais suportado. Passar um parâmetro time diferente de zero causará a falha da exclusão. Memcached::getResultCode() retornará MEMCACHED_INVALID_ARGUMENTS.

Valor Retornado

Retorna o novo valor do item em caso de sucesso ou false em caso de falha. O Memcached::getResultCode() retornará Memcached::RES_NOTFOUND se a key não existir.
