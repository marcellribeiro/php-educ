SessionHandler::destroy

(PHP 5 >= 5.4.0, PHP 7, PHP 8)

SessionHandler::destroy — Destrói uma sessão

Descrição

public SessionHandler::destroy(string $session_id): bool

Destrói uma sessão. Chamada internamente pelo PHP com session_regenerate_id() (desde que $destroy seja true), por session_destroy() ou quando session_decode() falhar.

Este método envolve o manipulador de gravação interno do PHP, definido na configuração ini session.save_handler, configurado antes da definição deste manipulador por session_set_save_handler().

Se esta classe for estendida por herança, chamar o método destroy da classe pai invocará a função que envolve este método, portanto invocará o callback interno. Isto permite que esse método seja substituído e ou interceptado e filtrado.

Para mais informações relacionadas ao que é esperado que este método faça, por favor leia a documentação de SessionHandlerInterface::destroy().

Parâmetros

session_id

O ID de sessão que estiver sendo destruído.

Valor Retornado

O valor retornado (geralmente true em caso de sucesso, false em caso de falha). Note que este valor é retornado internamente ao PHP para processamento.
