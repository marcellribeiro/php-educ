Como ajudar a melhorar a documentação

Há várias maneiras de ajudar a melhorar a documentação.

Se um erro for encontrado neste manual, em qualquer linguagem, por favor informe-os usando o rastreador de problemas (issues) do respectivo repositório de linguagem em » https://github.com/php. Por exemplo, erros no manual em inglês devem ser reportados em » https://github.com/php/doc-en/issues. Todos os problemas relacionados com a documentação, inclusive sobre os formatos de manuais, devem ser enviados como relatórios de bugs.

Nota:

Por favor não abuse dos rastreadores de problemas enviando pedidos de ajuda. Neste caso, use uma das muitas » opções de suporte.

Ao contribuir com notas, os usuários podem fornecer exemplos adicionais, ressalvas e esclarecimentos para outros leitores. Mas por favor, não informe bugs usando o sistema de anotações. Para mais detalhes, veja a seção intitulada 'Sobre as notas de usuários'.

Também é possível enviar pull requests para o » repositório espelho Github da documentação.

O manual PHP é traduzido em várias línguas. Saber inglês e uma língua estrangeira permite uma outra maneira de ajudar a melhorar o manual PHP trabalhando com um time de tradução. Para mais informações sobre como iniciar uma nova tradução, ou ajudar no projeto de tradução atual, por favor leia » http://doc.php.net/tutorial/.

O Projeto de Documentação do PHP também possui um canal IRC onde muitos autores do manual se reunem. Passe no canal #php.doc em irc.efnet.org e discuta maneiras de melhorar a documentação do PHP.
